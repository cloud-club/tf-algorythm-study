## 1. 🕑 Introduction

코딩테스트에 당당하게 응시할 수 있는 클둥이가 되자! ⛅️

<시즌 1 - 알고리즘 기초 다지기>

1. 미리 공지한 알고리즘 주제에 대해 공부해옵니다.
2. 주 1회 오프라인으로 모여 함께 제한 시간동안 문제를 풀기
3. 자신의 풀이를 팀원들에게 설명합니다.

<시즌 2 - 알고리즘 실전 역량 기르기>

1. 각종 기업 및 자격증 시험 기출 문제를 조사합니다.
2. 주 1회 오프라인으로 모여 함께 제한 시간동안 문제를 풉니다.
3. 자신의 풀이를 팀원들에게 설명합니다.
4. 스터디 이후, 다음 주차 스터디까지 자신의 풀이와, 개선한 풀이에 대해 설명하는 문서를 작성하여 PR을 제출합니다.

## 4. GitHub Collaboration Guidelines

### a. 디렉토리 구조

```text
.
├── README.md
└── week11
    ├── 1_퍼즐_게임_챌린지
    ├── 2_수식_복원하기
    └── README.md
```

### b. PR(Pull Request) 규칙

1. 브랜치 생성: `week00/영문이름`을 name으로 하는 브랜치 생성
2. 스터디 작업: 문제 별로 `./week00/1_문제_이름/영문이름.md`를 name으로 하는 파일 생성
3. PR 생성: `[Week00] 1 - 문제 이름`을 title로 하는 PR 생성
